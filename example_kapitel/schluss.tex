\chapter*{Schlussfolgerung und Ausblick in die Zukunft}

\addcontentsline{toc}{chapter}{Schlussfolgerung}

Das Projekt wurde vollständig abgeschlossen und zusätzlich ein Web Service eingerichtet. Dieser Web Service wurde im Hinblick auf einen möglichen Enterprise Service Bus erstellt. Wahrscheinlich zum Ende dieses Jahres wird Vodafone Technik einen ESB einführen, an welchem Systeme mit bestimmten Services angeschlossen werden können. Über diesen Bus kann dann Datenaustausch erfolgen. Bestimmte Systeme können Daten anbieten oder Daten konsumieren. Es werden zur Zeit die importierten Ticketdaten aus TTWOS als Web Service angeboten. Das bedeutet, dass der Web Service nur die Ticketnummer als Argument übergeben bekommt und dem anfragenden Klienten die Daten als XML-File über SOAP zur Verfügung stellt. Dies soll zeigen, dass das System AiO ohne großen weiteren Aufwand an den ESB angeschlossen werden kann. Weiterhin ist es möglich, über einen weiteren Web Service Dienst Daten in AiO zu speichern. So muss dann z.B. zukünftig nicht mehr ein direkter Datenbankzugriff auf TTWOS erfolgen, sondern TTWOS könnte über einen Web Service die Ticketdaten in AiO speichern oder alternativ könnten die Ticketdaten über einen Web Service bei TTWOS oder ChaMPS abgefragt werden. Dies hätte den Vorteil, dass keine Daten mehr in AiO hinterlegt werden müssten. 

Der im Rahmen des Projektes entwickelte Importer ist in der Lage, Daten aus vielen verschiedenen Datenbanken zu importieren. Dies ist auch leicht konfigurierbar mit Hilfe eines XML-Files, so dass mit geringem Aufwand Daten aus einer entfernten Datenbank in eine andere Datenbank importiert werden können. Der Vorteil dieses Ansatzes ist, dass dieses Importermodul auch für weitere Importvorgänge eingesetzt werden kann. Der Import kann entweder automatisiert oder manuell erfolgen. Auch hier hat der Benutzer bzw. Entwickler die Wahl. Der Importer wird ebenfalls schon für einen Import von Configuration Items aus C6000 erfolgreich verwendet. 

Es wurden in diesem Projekt neue Technologien eingesetzt. Nicht nur ein Web Service zählt zu den Technologien der Zukunft, sondern auch andere Techniken wie z.B. AJAX. Gut eingesetzt kann AJAX die Benutzeroberflächen angenehm, übersichtlich und auch schnell gestalten. Im Falle dieses Projektes wird AJAX dazu benutzt im Hintergrund Daten aus der Datenbank abzufragen, während der Benutzer noch eine Ticketnummer eintippt. 

Die Mitarbeiter der Abteilung TBAN können nun auf die Ticketdaten aus TTWOS und ChaMPS im abteilungsinternen Tool AiO zugreifen. Dies ist eine enorme Zeitersparnis und steigert die Qualität der Arbeit sowie die Effizienz des Fehlersuchprozesses. Ein weiterer Vorteil ist, dass mit dem Importmodul relativ schnell weitere Daten aus anderen Datenbanken importiert werden können. 

Die Planung des Projektes stellte sich größtenteils als richtig heraus. Änderungen am Projektplan sind üblich, gerade über einen Zeitraum von fünf Monaten. Unvorhergesehende Dinge können jeden Tag passieren, sei es, dass es Probleme mit der Hardware oder externen Systemen gibt oder das ein Vorgesetzter, dessen Unterschrift für einen Antrag benötigt wird für längere Zeit verreist ist. Solche Dinge sind nicht vorherzusehen und hier gilt es, die Planung mit einem Puffer zu versehen, der nicht zu groß ist, dass er das Projekt unötig in die Länge zieht und nicht zu klein ist, dass man möglicherweise bei einer kleinen Verzögerung schon in Verzug gerät. 

Das Importermodul könnte erweitert werden, so dass sämtliche Daten via Webinterface einzugeben sind. Damit ist gemeint, dass der Tabellenname, die zu importierenden Attribute sowie die Zugangsdaten zu der entsprechenden Datenbank einzugeben sind und anschließend ein Konfigurations-XML-File erstellt wird. Ein Verbindungstest soll sicherstellen, dass die Daten korrekt sind. Ein weiterer Verbesserungsvorschlag ist, eine Verbindung zur Quelldatenbank herzustellen, die Datenbanken sowie Tabellen anzeigt, anschließend auszuwählen, welche Tabellen aus welcher Datenbank importiert werden sollen, sowie die zu importierenden Attribute auszuwählen. Dies war bereits angedacht, ließ sich jedoch nicht realisieren, da kein Lesezugriff auf die Systemobjekte der SyBase-Datenbank besteht. Somit ist es nicht möglich, automatisch die Tabellennamen zu erfassen. 

Die Verbesserungen könnten vom Umfang und vom Wissenslevel her als eine Praktikumsaufgabe für einen Studenten des fünften Semesters dienen. Möglicherweise ließe sich gegen Ende des Jahres diese Aufgabe noch mit der Integration des AiO Web Services in den zukünftigen ESB kombinieren, so dass es vielleicht sogar eine Folgediplomarbeit geben könnte. 

Eine mögliche zusätzliche Erweiterung könnte die Umstellung des Systems AiO auf AJAX-Technologie sein. Die ohnehin schon gute und übersichtliche Software würde dadurch noch übersichtlicher, leichter handzuhaben und visuell ansprechender werden. Besonders Suchmasken und Filter sollten mit Hilfe von AJAX-Technologie deutlich performanter werden. Eine mögliche Studie könnte die Eigenschaften und Performance der GUI vorher und nachher miteinander vergleichen und eine Aussage über den Einsatz von AJAX in ITIL-Software treffen.
